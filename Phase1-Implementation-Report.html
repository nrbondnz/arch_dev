1. Executive Summary

The ARCH Sub-Contractor Management System is a custom-built web application designed to manage the full lifecycle of a subcontractor's work on a building project — from an initial enquiry, through quoting, daily progress, variations, progress claims, completion, and retention release. The primary user is the brick and stone crew, with supporting views for the main contractor and quantity surveyor.

Phase 1 focused on the foundations: getting sub-contractors, main contractors, and quantity surveyors set up with secure logins, and delivering the complete quoting engine — the ability to create a job, build a detailed quote line by line, submit it for approval, and have the main contractor accept or decline it, with the system automatically moving the job forward.

Key outcome: The core quoting workflow is fully operational end-to-end. A sub-contractor can now create a job and build a quote in the browser. When submitted, the main contractor sees it in their dashboard and can approve or decline it. On acceptance, the job automatically moves to "Contracted" status with the agreed value locked in. This replaces the current email/paper-based process for quoting.

The system is deployed on AWS (Amazon Web Services) and accessible through a standard web browser. User accounts are managed through industry-standard secure sign-in. Sample data can be generated on demand for demonstrations and testing.

2. What We Built

Phase 1 delivered five feature areas, all live and working against real data stored in the cloud.

2.1 Signing In & User Roles

Secure login with role-based views

Users sign up and log in using their email address. The system supports three distinct roles, and each sees a tailored view appropriate to their responsibilities:

  • Sub-contractor — the primary user. Your brick and stone crew. Sees jobs, quotes, and the tools for getting work done and paid.
  • Main Contractor — the builder you work for. Sees a list of submitted quotes and can approve or decline them.
  • Quantity Surveyor (QS) — the person who reviews and values work claims. Gets a dashboard tuned for review and approval tasks.

User roles are held centrally on the sign-in system so that each person automatically lands on the correct dashboard after logging in, with no manual selection required.

2.2 Managing Jobs

Capturing and tracking jobs through their lifecycle

Sub-contractors can create a new job at any time with the key details: job name, client, location, and a description of the work. Every job moves through a strict lifecycle that the system enforces automatically:

  • Enquiry — new job, no quote yet submitted
  • Quoted — a quote has been sent to the main contractor
  • Contracted — quote has been accepted, work is agreed

A filtered job list makes it easy to see only jobs in a given state — for example, all jobs awaiting a quote response. Tapping into a job opens a detail view organised into tabs: Overview, Quotes, Variations, and Claims. The Overview and Quotes tabs are live against the database; Variations and Claims are previewed with example data and will be wired up in Phase 2.

The system also protects the integrity of your data: contract values cannot be negative, and jobs cannot skip forward in the lifecycle (for example, a job cannot be marked Contracted unless it has an accepted quote).

2.3 The Quoting Engine Primary Deliverable

Line-by-line quote building with automatic totals

This is the heart of Phase 1 and the most substantial piece of functionality delivered. Sub-contractors can now build a fully-formed quote in the browser, replacing the current email/spreadsheet approach.

Building the quote:

  • Select an existing job, or create a new job right from the quote builder
  • Give the quote a title (e.g. "Face brickwork — Stage 1")
  • Add as many line items as needed, each with: description, unit of measure, quantity, and rate
  • Units supported: square metres (m²), linear metres (lm), number (no.), hours (hrs), and lump sum
  • The total for each line calculates automatically (quantity × rate)
  • The overall running subtotal, GST (15%), and total including GST update live as you go

Protecting your position:

  • A dedicated Exclusions & Assumptions field lets you clearly call out what is not included and what assumptions your price is based on
  • A Notes field for any additional context for the main contractor
  • Every quote has a validity period (default 30 days). After that, the system will refuse to accept an expired quote — forcing a fresh conversation rather than a surprise cost dispute

Review before submitting:

  • At any point you can save the quote as a draft and come back to it later
  • Before final submission, a review screen summarises the full breakdown: every line, the exclusions, the notes, the subtotal, the GST, and the total — so nothing goes out without a final check

What happens when you submit:

  • The quote status moves from Draft to Submitted
  • The associated job automatically moves from Enquiry to Quoted
  • The submission date is time-stamped for the validity period to run from
  • The quote now appears in the main contractor's dashboard for review

2.4 Quote Approval (Main Contractor / QS)

Approve or decline with comments — automatic downstream effects

When a main contractor logs in, they are taken directly to a dedicated approval dashboard. The Quotes tab shows all submitted quotes grouped by status (Submitted, Accepted, Rejected), with summary counts at the top.

For each pending quote, two actions are available:

  • Approve — with an optional comment. On approval, the system performs several checks and updates in a single, safe operation:
    • Confirms the quote has not expired (validity period still active)
    • Confirms no other quote for the same job has already been accepted (only one can be)
    • Moves the quote to Accepted status with the acceptance date recorded
    • Automatically advances the job to Contracted status
    • Locks in the quote total as the agreed contract value on the job
    If any step in this sequence fails, the whole operation is rolled back so the data stays consistent.
  • Decline — with a required reason. The quote moves to Rejected and the reason is captured.

This single feature replaces what is currently a back-and-forth of emails, phone calls, and manual contract value tracking.

2.5 The Sub-contractor Dashboard

Everything at a glance

When a sub-contractor logs in, they see a dashboard with:

  • A personalised greeting using their name
  • Summary tiles showing the number of active jobs and the number of quotes pending main contractor response — both live from the database
  • Quick-action buttons for the most common tasks: New Quote, Log Progress, Raise Variation, Submit Claim
  • Pull-to-refresh to fetch the latest data at any time

2.6 Demonstration & Testing Tools

Sample data generator

For demonstrations, training, and testing, a single button creates a realistic set of sample jobs in different states — including jobs still at enquiry, jobs with quotes submitted, and jobs already contracted with draft claims in progress. This gives new users something meaningful to explore without needing to type everything from scratch.

5. What Comes Next

With the quoting foundation in place, the next phases add the day-to-day site management features and the billing engine. The broad direction is to keep each phase producing something the business can use, rather than building silently in the background.

Phase 2 — Daily Progress & Documents

  • Daily Progress Log — record labour hours, work completed, materials used, delays, and site photos for each day of work on each job. This is the evidence base for later claims.
  • Document & photo upload — attach drawings, RAMS, contracts, and site photos to jobs
  • Notifications — first round of email alerts (quote submitted, quote accepted/rejected)

Phase 3 — Variations

  • Raise a variation — capture a scope change with reason, description, price, and programme impact
  • Variation approval — main contractor / QS approve or decline variations with comments
  • Automatic contract value update — on approval, the variation flows into the contract value and tags to appear in the next claim

Phase 4 — Progress Claims & Billing

  • Monthly claim builder — auto-populated from daily logs and approved variations, with manual adjustment
  • Retention calculation — tiered retention applied automatically
  • QS review workflow — claim submitted, reviewed, approved (or adjusted), and marked paid
  • PDF generation — automatic claim document generated and attached

Phase 5 — Full Job Lifecycle & Completion

  • Mobilisation checklist — RAMS, insurance, induction, materials, scaffolding tracked before work starts
  • Practical completion & defects — walk-through, defect tracking, sign-off
  • Retention release — partial at practical completion, final after defects period

Phase 6 — Polish & Integrations

  • Search — find anything across jobs, sites, and documents
  • Accounting integration — optional connector to Xero or MYOB
  • Offline support — daily logs work on poor site connectivity and sync when reconnected
  • Production hardening — monitoring, alerting, deployment pipeline

6. Technical Summary (Appendix)

For reference, the following is a non-technical summary of how the system is put together. None of this requires action from the business owner, but it's included for transparency and to support future conversations with auditors, insurers, or prospective partners.

Technology

  • Front end (what the user sees): Flutter — a modern Google framework that lets us build one application that runs in any web browser and, in future, can also be packaged as an iOS or Android app from the same codebase.
  • Back end (where the data lives): AWS Amplify Gen 2 — Amazon's managed cloud application platform. All data is stored in Amazon DynamoDB, a reliable cloud database used by many large organisations.
  • Sign-in security: Amazon Cognito — the same industry-standard sign-in service used by many major applications.
  • Hosting: AWS Amplify Hosting — automatically deploys new versions and serves the application globally with high availability.

How the Data Is Organised

The system's data is structured around a simple, clear model:

  • A Job is the top-level record and holds all work for a single project.
  • Each Job contains one or more Quotes (though only one can ever be accepted).
  • Each Quote contains multiple Line Items describing the work and price.
  • Each Quote can have one or more Claims against it as work progresses (the structure is in place ready for Phase 4 to build on).

Security & Access Control

Data access is enforced at the database level, not just in the application — this matters because it means even a bug in the app cannot accidentally expose data to the wrong user:

  • Sub-contractors can only see and edit their own jobs and quotes
  • Main contractors can read their sub-contractors' jobs and quotes, and update the status of quotes they approve
  • Quantity surveyors have read-only access for review purposes
  • All user passwords are managed and stored by Amazon Cognito, not by us

Reliability

All critical operations (like accepting a quote, which touches both the quote and the job) are written to roll back safely if any step fails — so the data stays consistent even if something goes wrong mid-operation.
